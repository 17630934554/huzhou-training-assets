# 物流仓库安环巡检监控系统方案

> 贝泰妮集团 · 湖州仓设备组  
> 2026年6月

---

## 一、项目背景

随着仓储作业量增长和安全合规要求提升，传统人工巡检模式存在以下痛点：

| 痛点 | 说明 |
|------|------|
| **覆盖不足** | 人工巡检无法24小时覆盖，夜间/节假日为盲区 |
| **响应滞后** | 隐患发现到上报再到处置，链路长、耗时久 |
| **记录缺失** | 纸质记录易丢、难查、无法统计分析和趋势追踪 |
| **责任模糊** | 整改不闭环，缺乏自动跟踪和超期提醒 |

**行业趋势**：AI视觉检测技术已成熟落地，抖音等平台广泛传播的"飞常检"等系统验证了AI巡检在仓储场景的可行性。本方案基于同等技术路线，针对物流仓库场景定制。

### 核心设计原则

1. **视频流不出内网** — 仅告警截图+短视频片段通过4G发出，安全合规
2. **边缘推理** — AI模型在本地盒子运行，不依赖云端算力，延迟<500ms
3. **轻量出站** — 4G仅传输告警数据（每月<50MB），成本可控
4. **双网隔离** — 内网取流 + 4G出站，互不干扰

## 四、飞书联动方案

### 4.1 告警卡片推送

告警按优先级推送不同样式卡片到飞书群：

- **P0 紧急**：红色卡片 + 关键信息 + 证据截图 + 整改倒计时
- **P1 重要**：橙色卡片 + 场景描述 + 证据截图
- **P2 一般**：仅写入多维表格，不推群（避免刷屏）

### 4.2 多维表格记录

已创建飞书多维表格「安环巡检记录」，包含13个字段：

编号（自动）→ 隐患类型 → 优先级 → 发现时间 → 监控点位 → 证据截图 → 证据视频 → 来源 → 责任人 → 整改状态 → 整改截止 → 整改照片 → 整改说明

访问地址：`https://xdp7r8eri5.feishu.cn/base/WNsFbGQEOaIANusrozfcY43Hn8b`

### 4.3 巡检看板

实时看板网页，展示今日告警总数、P0占比、整改率、摄像头状态、24h告警趋势、隐患类型分布等。

访问地址：`https://huzhou-training.pages.dev/inspection-dashboard.html`

## 八、安全与合规

1. **视频数据不出内网** — 仅告警截图+5秒短视频片段出站，原始视频流留存本地NVR
2. **4G通道加密** — 全程HTTPS传输，飞书API使用TLS1.2+
3. **权限最小化** — 飞书应用仅申请多维表格读写权限，不获取通讯录等敏感权限
4. **数据留存** — NVR按现有策略留存（通常30-90天），告警记录永久保存在飞书多维表格
5. **防误报机制** — 置信度阈值55%+同场景防抖冷却60-600秒，避免告警轰炸
